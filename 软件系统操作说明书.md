# 图书借阅管理系统 — 软件系统操作说明书

---

## 一、系统概述

### 1.1 项目简介

图书借阅管理系统是一套基于 Flask + SQLAlchemy + Jinja2 的 Web 应用程序，面向学校图书馆业务场景，提供图书管理、用户管理、借阅归还、统计排行等核心功能。系统采用前后端不分离架构，通过浏览器访问即可完成全部操作。

### 1.2 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.9 及以上 |
| 数据库 | SQL Server（默认），支持 MySQL |
| 浏览器 | Chrome / Edge / Firefox 最新版 |

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| Web 框架 | Flask | 轻量级服务端框架 |
| ORM | SQLAlchemy | 数据库对象关系映射 |
| 模板引擎 | Jinja2 | 服务端动态页面渲染 |
| 数据库驱动 | pyodbc / pymysql | SQL Server / MySQL 连接 |
| 前端 | HTML + CSS | 响应式布局，适配桌面端 |

## 二、安装与部署

### 2.1 前置准备

**安装 Python：**

1. 前往 [python.org](https://www.python.org/downloads/) 下载 Python 3.9+
2. 安装时勾选「Add Python to PATH」
3. 打开终端验证：`python --version`

**安装 SQL Server 驱动：**

- 下载 [ODBC Driver 17 for SQL Server](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server)
- 双击安装，全部默认即可

### 2.2 创建虚拟环境并安装依赖

```bash
# 进入项目目录
cd BooksManagementSystem

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

### 2.3 准备数据库

系统**不自动建表**，需在 SQL Server 中执行建库脚本：

1. 打开 **SQL Server Management Studio (SSMS)**
2. 连接到本地 SQL Server 实例
3. 打开文件 `SQL Server数据库设计.txt`
4. 执行全部脚本（F5）

脚本将自动完成：
- 创建数据库 `LibraryManagementSystem`
- 创建 5 张表（Category、Book、User、BorrowRecord、Admin）
- 创建索引、触发器、存储过程
- 插入基础数据（5 分类、15 本图书、7 名用户、1 管理员、6 条借阅记录）

### 2.4 配置数据库连接

编辑 `config.py`，确认连接字符串与本地环境匹配：

```python
# 默认：SQL Server Windows 身份验证
SQLALCHEMY_DATABASE_URI = (
    'mssql+pyodbc://@localhost/LibraryManagementSystem?'
    'driver=ODBC+Driver+17+for+SQL+Server&'
    'trusted_connection=yes&'
    'TrustServerCertificate=yes'
)
```

如果 SQL Server 实例名不是默认的 `localhost`，请修改为实际实例名（如 `localhost\SQLEXPRESS`）。

### 2.5 启动应用

```bash
python app.py
```

终端输出类似以下信息表示启动成功：

```
 * Running on http://127.0.0.1:5000
```

在浏览器中访问 **http://127.0.0.1:5000** 进入系统首页。

1. 在用户列表页点击「添加用户」
2. 填写姓名、密码
3. 选择用户类型（教师/研究生/本科生）
   - 下拉框会显示每种类型的配额信息
   - 选择后系统自动设置对应的借阅数量和期限
4. 点击「保存」

**用户类型配额对照：**

| 类型 | 最大可借 | 借阅期限 | 日违约金 |
|------|----------|----------|----------|

| 研究生 | 15 本 | 45 天 | 0.30 元 |
| 本科生 | 10 本 | 30 天 | 0.20 元 |

#### 编辑用户

1. 在用户列表中点击目标用户的「编辑」
2. 修改姓名、密码（留空不修改）、用户类型
3. 如需调整配额，直接修改「最大借阅数量」「借阅期限」「日违约金」
4. 点击「保存」

#### 删除用户

1. 在用户列表中点击目标用户的「删除」
2. 确认删除操作

**限制：** 有未归还图书的用户无法删除，需先归还所有图书。

### 3.4 图书管理

> 需要管理员权限

#### 查看图书列表

点击导航栏「管理」→「图书管理」，进入图书列表页面。

- 按分类筛选图书
- 搜索书名
- 查看每本书的库存状态（总库存 / 可借数量）

#### 添加图书

1. 点击「添加图书」
2. 填写 ISBN、书名、作者、出版社、出版年份
3. 选择分类，填写总库存和架位号
4. 点击「保存」

#### 编辑图书

1. 在图书列表中点击目标图书的「编辑」
2. 修改需要变更的字段
3. 点击「保存」

#### 删除图书

1. 在图书列表中点击目标图书的「删除」
2. 确认删除操作

**限制：** 有借出记录的图书无法删除。

### 3.5 分类管理

> 需要管理员权限

#### 查看分类列表

点击导航栏「管理」→「分类管理」。

- 显示每个分类的编号、名称、馆藏位置、备注

#### 添加分类

1. 点击「添加分类」
2. 填写分类号（如 TP3）、分类名称、馆藏位置
3. 点击「保存」

#### 编辑 / 删除分类

- 编辑：修改分类名称、馆藏位置、备注
- 删除：只能删除没有关联图书的分类

### 3.6 借阅图书

#### 操作步骤

1. 点击导航栏「借阅图书」或首页「立即借阅」
2. 在表单中输入：
   - **用户 ID**：借书人的编号（可在用户管理页面查看）
   - **图书 ID**：要借的图书编号（可在图书管理页面查看）
3. 点击「确认借阅」

#### 系统自动校验

- 用户是否存在
- 图书是否存在
- 用户是否已达借阅上限
- 图书是否还有库存

#### 校验通过后

- 系统自动计算应还日期（根据用户类型）
- 图书可借库存 -1
- 用户已借数量 +1
- 显示借阅成功和应还日期

#### 快速查找辅助

如果不确定用户 ID 或图书 ID，可使用「快速查找」功能：

1. 点击导航栏「快速查找」
2. 输入书名、ISBN 或用户姓名
3. 系统返回匹配的图书和用户列表
4. 记录需要使用的 ID

### 3.7 归还图书

#### 操作步骤

1. 点击导航栏「归还图书」或首页「立即归还」
2. 输入借阅记录 ID（可在「借阅记录」页面查看）
3. 点击「确认归还」

#### 系统自动处理

- 记录实际归还日期
- 图书可借库存 +1
- 用户已借数量 -1
- 超期归还自动计算违约金

**违约金计算规则：**

```
违约金 = (实际归还日期 - 应还日期) × 日违约金标准
```

例如：本科生借书，应还日期 7月15日，实际归还 7月18日，超期 3 天，日违约金 0.20 元，违约金 = 3 × 0.20 = 0.60 元。

### 3.8 借阅记录查询

点击导航栏「借阅管理」→「借阅记录」。

- 查看所有借阅历史（借出中 + 已归还）
- 按用户 ID 筛选
- 按状态筛选（借出中 / 已归还）
- 每条记录显示：图书名、用户名、借阅日期、应还日期、归还日期、违约金、状态

### 3.9 超期记录

点击导航栏「借阅管理」→「超期记录」。

- 显示所有超期未还的借阅记录

## 五、常见问题

### Q1：启动报错 "Can't open lib 'ODBC Driver 17 for SQL Server'"

**原因：** 未安装 SQL Server ODBC 驱动。

**解决：** 下载安装 [ODBC Driver 17 for SQL Server](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server)。

### Q2：统计页面报错 "列没有包含在聚合函数或 GROUP BY 子句中"

**原因：** 使用了旧版本代码，GROUP BY 不完整。

**解决：** 确保使用最新代码，已修复 GROUP BY 子句。

### Q3：用户无法删除

**原因：** 该用户仍有未归还的图书。

**解决：** 先在「归还图书」页面归还所有图书，再删除用户。

### Q4：图书无法删除

**原因：** 该图书有借阅记录（包括已归还的）。

**解决：** 联系管理员，或先删除关联的借阅记录。

### Q5：如何切换数据库

**解决：** 修改 `config.py` 中的 `SQLALCHEMY_DATABASE_URI`：

```python
# MySQL
SQLALCHEMY_DATABASE_URI = 'mysql+pymysql://root:password@localhost/LibraryManagementSystem?charset=utf8mb4'

# 或通过环境变量
set DATABASE_URL=mysql+pymysql://root:password@localhost/LibraryManagementSystem?charset=utf8mb4
```

### Q6：忘记管理员密码

**解决：** 在 SQL Server 中执行：

```sql
UPDATE Admin SET password = 'new_password' WHERE username = 'admin';
```

## 七、安全性与健壮性

### 7.1 输入验证

系统对所有用户输入进行严格验证，防止非法输入导致系统崩溃：

- **ID 参数验证**：所有 ID 参数必须为正整数，非数字输入会返回友好错误提示
- **必填字段验证**：表单提交时验证必填字段，防止空值提交
- **业务规则验证**：借阅时验证库存和配额，归还时验证记录归属

### 7.2 权限控制

- **用户权限隔离**：普通用户只能查看和操作自己的借阅记录，不能访问管理功能
- **管理员权限保护**：管理功能需要管理员登录，使用 `@admin_required` 装饰器保护
- **越权操作防护**：归还图书时验证记录归属，防止用户归还他人借阅记录

### 7.3 异常处理

- **数据库事务管理**：所有数据库操作使用 try-except 包裹，失败时自动回滚
- **友好错误提示**：异常发生时返回用户友好的错误信息，不暴露技术细节
- **日志记录**：使用 logger 记录异常堆栈，便于问题排查

### 7.4 常见错误处理

| 错误场景 | 系统提示 | 处理方式 |
|----------|----------|----------|
| 输入非数字 ID | "请输入有效的ID" | 返回表单重新输入 |
| 借阅记录不存在 | "借阅记录不存在" | 提示用户检查 ID |
| 图书已归还 | "该图书已归还" | 提示用户检查记录状态 |
| 数据库操作失败 | "操作失败，请稍后重试" | 自动回滚事务 |
