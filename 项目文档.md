# 图书借阅管理系统 — 项目文档

## 一、项目概述

### 1.1 项目背景

本项目为数据库课程实践「图书借阅管理系统」的软件开发部分，严格遵循 Jinja 分支技术路线，采用 **Flask + Jinja2** 前后端不分离架构。系统通过 Python 高级语言实现数据库连接与操作，封装图书借阅核心业务逻辑，开发可动态交互的可视化界面，完整支撑图书馆馆藏管理、借阅管控、统计排行等业务场景。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 数据库连接与操作 | 通过 SQLAlchemy ORM 以 Python 语法替代原生 SQL，实现跨数据库适配 |
| 核心业务逻辑封装 | 借阅权限校验、超期违约金自动计算、库存同步、热度统计算法 |
| 可视化交互界面 | 基于 Jinja2 模板引擎实现动态页面渲染与表单交互 |

### 1.3 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows / Linux / macOS |
| Python | 3.9+ |
| 数据库 | SQL Server（需提前执行 SQL Server数据库设计.txt 建库建表） |
| 浏览器 | Chrome / Edge / Firefox 等现代浏览器 |

## 四、数据库设计

### 4.1 E-R 模型

系统包含 5 张核心业务表，实体关系如下：

- **Category**（分类） 1:N **Book**（图书）
- **User**（用户） 1:N **BorrowRecord**（借阅记录）
- **Book**（图书） 1:N **BorrowRecord**（借阅记录）
- **Admin**（管理员）独立实体

> 注：用户类型已合并至 User 表，不再有独立的 UserType 实体。

### 4.2 表结构详情

#### Category（图书分类表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| cat_id | CHAR(10) | PK | 中图法分类号（如TP3） |
| cat_name | NVARCHAR(50) | NOT NULL | 分类名称 |
| location | NVARCHAR(100) | | 馆藏位置 |
| remark | NVARCHAR(200) | | 备注 |

#### Book（图书表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| book_id | INT | PK, AUTO_INCREMENT | 图书编号 |
| isbn | VARCHAR(20) | UNIQUE, NOT NULL | ISBN编号 |
| title | NVARCHAR(200) | NOT NULL | 书名 |
| author | NVARCHAR(100) | | 作者 |
| publisher | NVARCHAR(100) | | 出版社 |
| pub_year | INT | | 出版年份 |
| cat_id | CHAR(10) | FK → Category | 分类编号 |
| total_stock | INT | NOT NULL, CHECK ≥0 | 总库存 |
| available_stock | INT | NOT NULL, CHECK ≥0 | 可借库存 |
| shelf_location | NVARCHAR(50) | | 架位号 |

#### User（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | INT | PK, AUTO_INCREMENT | 用户编号 |
| name | NVARCHAR(50) | NOT NULL | 姓名 |
| password | NVARCHAR(50) | NOT NULL | 登录密码 |
| user_type | NVARCHAR(20) | NOT NULL | 教师 / 研究生 / 本科生 |
| max_borrow | INT | NOT NULL | 最大可借数量 |
| days_limit | INT | NOT NULL | 借阅期限（天） |
| penalty_per_day | DECIMAL(10,2) | NOT NULL | 超期日违约金（元） |
| borrowed_count | INT | DEFAULT 0 | 当前已借数量 |
| penalty_total | DECIMAL(10,2) | DEFAULT 0 | 累计违约金 |

#### BorrowRecord（借阅记录表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| record_id | INT | PK, AUTO_INCREMENT | 记录编号 |
| book_id | INT | FK → Book | 图书编号 |
| user_id | INT | FK → User | 用户编号 |
| borrow_date | DATE | NOT NULL | 借阅日期 |
| due_date | DATE | NOT NULL | 应还日期 |
| return_date | DATE | NULL | 实际归还日期 |
| penalty | DECIMAL(10,2) | DEFAULT 0 | 违约金 |
| status | TINYINT | DEFAULT 0 | 0=借出中, 1=已归还 |

#### Admin（管理员表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| admin_id | INT | PK, AUTO_INCREMENT | 管理员编号 |
| username | VARCHAR(30) | UNIQUE, NOT NULL | 用户名 |
| password | VARCHAR(50) | NOT NULL | 密码 |

## 五、功能模块

### 5.1 用户管理模块

维护教师、研究生、本科生三类用户信息，绑定差异化借阅权限。

| 功能 | 路由 | 说明 |
|------|------|------|
| 用户列表 | GET `/users/` | 支持按姓名搜索，展示用户类型、已借数、违约金 |
| 添加用户 | GET/POST `/users/add` | 选择用户类型，系统自动关联权限 |
| 编辑用户 | GET/POST `/users/edit/<id>` | 修改姓名、密码、用户类型 |
| 删除用户 | POST `/users/delete/<id>` | 校验是否有未归还图书，防止误删 |

**权限控制**：仅管理员可进行增删改操作。

### 5.2 图书与分类管理模块

管理图书基础信息、中图分类、馆藏位置与实时库存。

| 功能 | 路由 | 说明 |
|------|------|------|
| 图书列表 | GET `/books/` | 支持按书名/作者搜索、按分类筛选 |
| 添加图书 | GET/POST `/books/add` | 录入 ISBN、书名、分类、库存等 |
| 编辑图书 | GET/POST `/books/edit/<id>` | 库存修改时校验不低於借出数量 |
| 删除图书 | POST `/books/delete/<id>` | 校验是否有未归还记录 |
| 分类列表 | GET `/categories/` | 展示所有分类及馆藏位置 |
| 添加分类 | GET/POST `/categories/add` | 新增中图分类 |
| 编辑分类 | GET/POST `/categories/edit/<id>` | 修改分类信息 |
| 删除分类 | POST `/categories/delete/<id>` | 校验分类下是否有图书 |

### 5.3 借阅归还核心模块

实现借书、还书全流程操作，是系统核心业务单元。

| 功能 | 路由 | 说明 |
|------|------|------|
| 借阅图书 | GET/POST `/borrow/` | 输入用户ID和图书ID，自动校验权限 |
| 归还图书 | GET/POST `/borrow/return_page` | 输入记录ID，自动计算超期违约金 |
| 借阅记录 | GET `/borrow/records` | 按用户ID/状态筛选，展示全部记录 |
| 超期未还 | GET `/borrow/overdue` | 列出所有超期未归还的图书 |
| 快速查找 | GET `/borrow/quick_search` | 按书名/作者/ISBN/姓名多条件检索 |

**借阅业务规则**：

1. 校验用户是否存在
2. 校验图书是否存在且库存充足（`available_stock > 0`）
3. 校验用户是否有剩余借阅额度（`borrowed_count < max_borrow`）
4. 自动计算应还日期：`borrow_date + days_limit`
5. 创建借阅记录（status=0），同步更新图书库存和用户已借数

**归还业务规则**：

1. 校验记录是否存在且未归还（status=0）
2. 设置归还日期和状态（status=1）
3. 恢复图书库存（`available_stock + 1`）
4. 减少用户已借数（`borrowed_count - 1`）
5. 若超期：`违约金 = (归还日 - 应还日) × 日违约金标准`，累加到用户总违约金

### 5.4 统计排行模块

统计单本图书借阅热度、分类借阅频率、用户借阅频度，生成多维度排行榜。

| 功能 | 路由 | 说明 |
|------|------|------|
| 综合看板 | GET `/stats/` | 馆藏/用户/借出/超期数量、TOP10 排行 |
| 图书排行 | GET `/stats/books` | 图书借阅热度排行，支持 TOP 5/10/20 |
| 分类排行 | GET `/stats/categories` | 分类借阅频率排行 |
| 用户排行 | GET `/stats/users` | 用户借阅频度排行，支持 TOP 5/10/20 |

### 5.5 管理员认证模块

| 功能 | 路由 | 说明 |
|------|------|------|
| 登录 | GET/POST `/login` | 管理员登录认证 |

## 六、安装部署

确保已安装 Python 3.9 或更高版本：

### 6.2 安装依赖


### 6.3 准备数据库

系统**不会自动建表**，需提前在 SQL Server 中执行 `SQL Server数据库设计.txt` 中的建库建表脚本，并插入基础数据。

1. 打开 SQL Server Management Studio (SSMS)
2. 执行 `SQL Server数据库设计.txt` 全部内容
3. 确认数据库 `LibraryManagement` 已创建，6 张表已生成

### 6.4 配置数据库连接

编辑 `config.py`，修改连接字符串以匹配你的 SQL Server 环境：

```python
# Windows 身份验证（默认）
SQLALCHEMY_DATABASE_URI = (
    'mssql+pyodbc://@localhost/LibraryManagementSystem?'
    'driver=ODBC+Driver+17+for+SQL+Server&'
    'trusted_connection=yes&'
    'TrustServerCertificate=yes'
)

# 用户名密码登录
SQLALCHEMY_DATABASE_URI = (
    'mssql+pyodbc://sa:your_password@localhost/LibraryManagementSystem?'
    'driver=ODBC+Driver+17+for+SQL+Server&'
    'TrustServerCertificate=yes'
)

# MySQL 连接
SQLALCHEMY_DATABASE_URI = (
    'mysql+pymysql://root:your_password@localhost/LibraryManagementSystem?charset=utf8mb4'
)
```

### 6.5 启动应用

```bash
python app.py
```

启动后访问 [http://127.0.0.1:5000](http://127.0.0.1:5000)。

### 违约金计算公式

```
违约金 = (实际归还日期 - 应还日期) × 日违约金标准
```

- 按期或提前归还：违约金 = 0
- 超期归还：按天累计，自动更新借阅记录和用户总违约金

### 库存同步规则

| 操作 | 图书 available_stock | 用户 borrowed_count |
|------|---------------------|---------------------|
| 借书 | -1 | +1 |
| 还书 | +1 | -1 |

## 九、数据库连接原理

### 9.1 架构说明

系统**不负责建表**，只负责**连接已有数据库并操作数据**：

```
┌──────────────┐     SQLAlchemy ORM      ┌──────────────────┐
│  Flask 应用   │ ◄──────────────────────► │  SQL Server 数据库 │
│  (app.py)    │   pyodbc 驱动连接         │  (已有库表+数据)   │
└──────────────┘                          └──────────────────┘
```

- **app.py**：Flask 应用入口，通过 `db.init_app(app)` 将 SQLAlchemy 绑定到 Flask
- **config.py**：配置数据库连接 URI（`mssql+pyodbc://...`），指向已有 SQL Server 实例
- **models.py**：ORM 实体类映射已有表结构，**不执行 DDL**（无 `db.create_all()`）
- **routes/**：路由层接收请求，调用 services 层，通过 ORM 操作数据库
- **services/**：业务逻辑层，封装借阅校验、违约金计算等核心规则

### 9.2 连接流程

1. 启动时 `db.init_app(app)` 读取 `config.py` 中的连接 URI
2. SQLAlchemy 通过 `pyodbc` 驱动连接 SQL Server
3. 请求到达时，Flask 创建请求上下文，SQLAlchemy 提供 `db.session` 数据库会话
4. 路由通过 `db.session.query()` / `db.session.add()` / `db.session.commit()` 操作数据
5. 请求结束时自动关闭会话，连接归还连接池

### 9.3 管理员登录逻辑

管理员登录认证的是**数据库中 Admin 表**的数据，而非本地配置：

```
用户输入 admin / admin123
       │
       ▼
auth.py: Admin.query.filter_by(username='admin', password='admin123').first()
       │
       ▼
SQLAlchemy 生成: SELECT * FROM Admin WHERE username='admin' AND password='admin123'
       │
       ▼
SQL Server 执行查询，返回匹配的管理员记录
       │
       ▼
匹配成功 → 写入 session → 登录成功
```

## 十、设计亮点

1. **三层架构清晰分离**：表现层（路由+模板）、业务逻辑层（services）、数据访问层（ORM），职责明确，易于维护和扩展。

2. **SQLAlchemy ORM 抽象**：完全通过 Python 语法操作数据库，修改 `config.py` 一行配置即可切换 SQL Server / MySQL，无需修改任何业务代码。

3. **借阅权限自动校验**：借书时自动根据用户类型检查剩余可借额度，防止超限借阅；还书时自动计算超期违约金，保证业务规则统一落地。

4. **多维度统计分析**：从图书热度、分类频率、用户活跃度三个维度生成排行榜，支持 TOP N 动态切换，支撑管理员运营分析。

5. **响应式界面设计**：基于 CSS Grid/Flexbox 的响应式布局，适配桌面端与移动端，导航栏含下拉菜单，交互流畅。

6. **数据完整性保护**：删除用户时校验是否有未归还图书，删除图书时校验是否有借出记录，删除分类时校验是否有关联图书，防止脏数据产生。
